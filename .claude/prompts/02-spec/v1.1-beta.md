## 🎯 Objetivo

Transformar o PRD em uma **SPEC técnica executável** que o desenvolvedor possa implementar sem adivinhar.

---

## 📋 Checklist Executivo

- [ ] PRD lido completamente
- [ ] Escopo (IN/OUT) definido
- [ ] Arquivos listados (criar/modificar)
- [ ] Fluxos documentados
- [ ] Testes mapeados
- [ ] Riscos identificados
- [ ] SPEC salva em `.claude/specs/`

## 1️⃣ Escopo

### ✅ Dentro (IN)
```yaml
files:
  - {lista de arquivos}
  - {lista de arquivos}
features:
  - {descrição}
  - {descrição}
```

### ❌ Fora (OUT)
```yaml
excluded:
  - {o que NÃO faz}
  - {o que NÃO faz}
```

---

## 2️⃣ Arquivos

### A Criar
```yaml
files:
  - path: "frontend/src/components/NewComponent.tsx"
    type: "React Component"
    responsibility: "UI para {descrição}"
    tests: "NewComponent.test.tsx"

  - path: "backend/src/routes/new-endpoint.ts"
    type: "Fastify Route"
    responsibility: "API para {descrição}"
    tests: "new-endpoint.test.ts"
```

### A Modificar
```yaml
changes:
  - file: "existing-file.ts"
    changes:
      - "Add function X"
      - "Modify endpoint Y"
    impact: "Breaking? No. Migration needed? No."
```
