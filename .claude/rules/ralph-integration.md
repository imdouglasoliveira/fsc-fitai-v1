# Ralph Integration — Detailed Rules

Status: **LIVE** (Integrated with Story Lifecycle Phase 2)

---

## Overview

Ralph é um **autonomous loop** que explora iterativamente histórias ambíguas até convergência.

Esta rule define:
- ✅ Quando Ralph é apropriado (vs SDD linear)
- ✅ Como Ralph respeita AIOS governance
- ✅ Escalation triggers
- ✅ Status transitions
- ✅ Worktree cleanup

---

## Decision Matrix: Ralph vs SDD

### Ralph is BETTER when:
```
Requirement Clarity      < 60%   (ambíguo, exploratório)
AND
Iteration Estimate       >= 3    (multiple passes needed)
AND
Time Pressure           LOW      (não é hotfix crítico)
AND
Complexity              8-20     (standard a complex)
```

### SDD is BETTER when:
```
Requirement Clarity      >= 80%  (bem-especificado)
OR
Time Pressure           HIGH     (< 4 horas)
OR
Complexity              <= 8     (muito simples)
OR
Story Type              = Hotfix (critical bug)
```

### HYBRID (Start SDD, switch to Ralph):
```
Story claridade começa em 70%
Mas durante Spec-writing, descobre-se que é complexa
→ Pode switchear: Draft Spec → Ralph Loop → Refine → Resume SDD
```

## Story Status Transitions with Ralph

```
Draft (initial)
  ↓
[Clarification step]
  ├─ Is it clear (>=80%)? → Yes → SDD path (Validate → Implement)
  └─ Is it ambiguous (<60%)? → Yes → Ralph path
       ↓
    Ralph Loop (autonomous iterations)
       ↓
    Converged? → Yes → Spec finalized
       ↓
    Story: Draft → Ready (normal SDD path)
       ↓
    @dev: Implement (same as SDD)
```

## Ralph Loop Configuration

### Max Iterations
```
Default:    5 iterations
Can change: In story config
If reached: Escalate (manual review needed)
```

### Iteration Timeout
```
Each iteration:  < 15 minutes
If longer:       Escalate (infinite loop detection)
Total Ralph:     < 90 minutes
```

### Worktree Lifecycle
```
Create:  When Ralph starts (.claude/worktrees/STORY-X-ralph/)
Monitor: Progress tracking in progress.json
Clean:   When converged OR escalated
Retain:  Only final spec (delete working files)
```

## Convergence Criteria

Ralph STOPS when ANY of these is true:

```
1. HAPPY CASE: Spec stable
   └─ Spec version hasn't changed >10% from previous
   └─ All AC (acceptance criteria) defined
   └─ DoD (definition of done) clear
   └─ Iteration count < 5

2. MAX ITERATIONS: Reached limit
   └─ Iteration count = 5
   └─ Save best attempt
   └─ Flag for manual review: @po or @architect

3. ERROR LOOP: Stuck
   └─ Same error 3 iterations in a row
   └─ Example: "Cannot integrate with OAuth provider"
   └─ Requires external decision
   └─ Escalate immediately

4. CLARITY IMPROVED: No longer needs Ralph
   └─ During Ralph, discovers: "Actually, this IS clear"
   └─ Switch to SDD (faster from here)
   └─ Cancel Ralph, continue as normal spec

5. EXTERNAL BLOCKER: New dependency found
   └─ "Need API from other team first"
   └─ Cannot proceed autonomously
   └─ Escalate with blockers documented
```

---

## Escalation Triggers

### IMMEDIATE (Block Ralph)
```
├─ External API not ready
├─ Missing key requirement (e.g., auth method undefined)
├─ Discovers critical architectural decision needed
└─ Security concern found
```

### AT MAX ITERATIONS (5/5)
```
└─ No convergence despite 5 attempts
   └─ Ralph saves best attempt
   └─ Flags for @po/@architect manual review
   └─ Story remains Draft (not Ready)
```

### ON ERROR LOOP (3x same error)
```
└─ Example: "Cannot write tests for hypothesis X"
└─ Ralph cannot progress
└─ Escalate with: error context + current spec state
```

## Integration with AIOS Governance

### @po (Product Owner) Role
```
BEFORE Ralph:
  - Review clarity score
  - Approve: "Yes, this is ambiguous. Ralph is appropriate"

AFTER Ralph:
  - Review final spec generated by Ralph
  - Validate: All AC met? DoD clear?
  - Decision: GO (Ready) or NO-GO (needs revision)
  - If NO-GO: Can loop Ralph again OR ask @dev to refine manually
```

### @architect Role
```
WHEN Ralph escalates (blocker found):
  - Review: What blocker was found?
  - Decide: Can be resolved? Or fundamental?
  - If resolvable: Provide guidance → Ralph continues
  - If fundamental: Design decision needed → Manual path
```

### @dev Role
```
WHEN Ralph completes:
  - Receives spec from Ralph (same as @po-approved SDD spec)
  - Implements normally (@dev path)
  - Zero difference from SDD path forward
```

### Decision Auditor Role
```
EVERY Ralph loop:
  - Record: Why was Ralph chosen?
  - Record: How many iterations took?
  - Record: Did it converge or escalate?
  - Learning: "Ralph worked well for X" or "Ralph didn't help for Y"
```

## Worktree Management

### Create
```bash
When:   Ralph initializes
Path:   .claude/worktrees/STORY-{id}-ralph/
Init:   progress.json created with iteration=0
```

### During Ralph
```
Iteration 1:
  ├─ prd-v1.md (PRD draft from Ralph)
  ├─ spec-v1.md
  └─ hypothesis-log.json

Iteration 2:
  ├─ prd-v2.md (refined)
  ├─ spec-v2.md
  └─ hypothesis-log.json (updated)

... (repeat until convergence)
```

### After Ralph (Cleanup)
```bash
If converged:
  → Copy final spec to .claude/specs/STORY-{id}-spec.md
  → Delete worktree (save space)
  → Keep progress.json (for decision log)

If escalated:
  → Keep worktree (preserve state for manual review)
  → Tag: "Requires manual review - [reason]"
  → @po or @architect takes over
```

## Interaction with QA Loop

```
Normal SDD → QA Loop (iterative review)
Ralph → Converged Spec → @dev implementation → QA Loop

Ralph doesn't replace QA Loop.
Ralph is PRE-DEV (clarifies requirements).
QA Loop is POST-DEV (validates code).
```

## Troubleshooting

### "Ralph is stuck in loop"
```
1. Check: What error repeating?
2. If 3x same: Escalate immediately (don't wait for 5 iterations)
3. Check blockers: External dependency? Design decision needed?
```

### "Ralph generated unclear spec"
```
1. This is OK. Ralph still helped narrow down ambiguity.
2. @po can validate: "Is this better than starting PRD?"
3. If better: Mark Ready, continue
4. If worse: Loop Ralph again OR switch to manual spec
```

### "Should have used Ralph, but didn't"
```
1. Story history shows: "Took 3 days to spec. Was iterative."
2. Learning: "Next similar story → use Ralph"
3. Update decision-log: "Would have saved 2 days with Ralph"
```

## Ralph ≠ Full Implementation

⚠️ **IMPORTANT:**
```
Ralph generates SPEC, not CODE.
Ralph loop is finished when:
  ✅ Spec is clear (AC, DoD defined)
  ✅ Architecture decided
  ✅ Not blocked by external factors

Then @dev implements code (normal process).
Code still goes through TDD + QA loop.
```
